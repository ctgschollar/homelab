# Subnet Scanner & Inventory Generator

**Date:** 2026-05-04
**Status:** Approved

## Overview

A Python script that scans a subnet for active hosts, interactively assigns hostnames and inventory groups, and writes a fresh Ansible inventory and CoreDNS zone file. The Ansible inventory is the source of truth for all infrastructure; the zone file is derived from it.

## Goals

- Discover hosts on a LAN subnet not yet in the Ansible inventory
- Interactively assign each a hostname (FQDN: `hostname.<domain>`) and inventory group
- Store MAC address per host (in inventory and as a zone file comment)
- Generate a clean `inventory.yml` on first run (user backs up the old one)
- On subsequent runs, only prompt for newly discovered hosts
- Regenerate `<domain>.db` zone file with explicit A records + configurable wildcard IPs
- Update the CoreDNS `Corefile` to remove the legacy `home.` zone stanza
- Configurable domain, subnet, and file paths via CLI flags

## Non-Goals

- Migrating or patching the existing inventory — user backs it up and starts fresh
- NetBox integration (future work)
- Ansible facts gathering (future work)

## File Structure

```
ansible/
  scripts/
    scan_subnet.py       ← new script
  pyproject.toml         ← add questionary, rich, python-nmap deps
coredns/
  zones/
    schollar.dev.db      ← regenerated by script
  Corefile               ← updated by script (home. stanza removed)
```

## CLI

```
hatch run python scripts/scan_subnet.py \
  [--subnet 192.168.3.0/24] \
  [--domain schollar.dev] \
  [--inventory ../inventory.yml] \
  [--zone-file ../coredns/zones/<domain>.db] \
  [--corefile ../coredns/Corefile]
```

All paths default to their standard locations relative to the script file. `--domain` defaults to `schollar.dev`.

**System requirement:** `nmap` must be installed on the host running the script. The script checks for it at startup and exits with a clear error if missing.

## Script Flow

### 1. Startup

- Resolve all file paths (defaults or overrides)
- Check `nmap` is available (`shutil.which`)
- Load existing inventory if present — build a set of known IPs to skip

### 2. Scan

Run `nmap -sn <subnet> -oX -` and parse the XML output. For each responding host, extract:
- IP address
- MAC address (available for hosts on the same L2 segment)
- Hostname (from nmap PTR/reverse DNS lookup, if any)

### 3. Diff

Filter out any IP already present in the inventory. Print a summary:
```
Scan complete: 24 hosts found, 13 already in inventory, 11 new
```
If no new hosts, exit cleanly.

### 4. Interactive Loop

For each new host (sorted by IP):

```
────────────────────────────────────────
Found: 192.168.3.xxx
  MAC: aa:bb:cc:dd:ee:ff
  nmap name: foo.local (if detected)

Hostname (leave blank to skip): claude
→ Will register as: claude.schollar.dev

Group:
  ❯ proxmox_vms
    proxmox_nodes
    swarm_nodes
    + Create new group
```

- Blank hostname → host is skipped and not added anywhere
- "Create new group" → prompt for group name; new group is created in the inventory

### 5. Confirm

Before writing any files, show a summary of all planned changes:

```
Ready to write:
  inventory.yml      +3 hosts across 2 groups
  schollar.dev.db    +3 A records  (serial: 2026050401 → 2026050402)
  Corefile           remove home. stanza

Proceed? [y/N]
```

On first run (zone file does not exist), the serial line shows: `serial: 2026050401 (new file)`.

### 6. Write

Write changes atomically (write to `.tmp`, then `os.replace`). Print a reminder:

```
Done. Redeploy the coredns stack via Portainer to apply DNS changes.
```

## Inventory Format

Each host entry written by the script:

```yaml
hostname:
  ansible_host: hostname.schollar.dev
  mac_address: aa:bb:cc:dd:ee:ff
```

Group-level `vars` blocks (`ansible_user`, `ansible_python_interpreter`, `ansible_ssh_private_key_file`) are preserved on subsequent runs. New groups get these defaults:

```yaml
vars:
  ansible_user: root
  ansible_python_interpreter: /usr/bin/python3
  ansible_ssh_private_key_file: "{{ inventory_dir }}/ansible_ssh_key"
```

On first run (no existing inventory), the script generates the full file from scratch including group structure.

## Zone File Format

Generated `<domain>.db`:

```
$ORIGIN schollar.dev.
$TTL 300
@   IN SOA ns.schollar.dev. hostmaster.schollar.dev. (
        20260504   ; serial
        3600       ; refresh
        600        ; retry
        604800     ; expire
        300 )      ; minimum

    IN NS  ns.schollar.dev.
ns  IN A   <first wildcard IP>

; --- Hosts
claude       IN A  192.168.3.79  ; aa:bb:cc:dd:ee:ff
edge01       IN A  192.168.3.91  ; bb:cc:dd:ee:ff:00
...

; --- Wildcard → Traefik
*   IN A  192.168.3.70
    IN A  192.168.3.71
    IN A  192.168.3.72
    IN A  192.168.3.73
@   IN A  192.168.3.70
```

**First run vs subsequent runs:** The script checks whether the zone file exists on disk.
- Zone file **does not exist** → full regeneration: prompt for wildcard IPs, write the complete file from scratch
- Zone file **exists** → append-only: new A records inserted before the wildcard block, serial incremented, wildcard block left untouched

**Serial management:**
- Full regeneration: `YYYYMMDD01` (e.g. `2026050401`)
- Append run: parse current serial from SOA, add 1 (e.g. `2026050401 → 2026050402`)

**Wildcard IPs:** Prompted once during full regeneration only:
```
Wildcard *.schollar.dev should point to which IPs? (your Traefik nodes)
Enter IPs one per line, blank to finish:
> 192.168.3.70
> 192.168.3.71
> 
```

## Corefile Update

The script removes the `home.:53 { ... }` stanza from the Corefile. The `home.db` zone file is left on disk (not deleted) — the user can remove it manually once satisfied.

## Error Handling

| Situation | Behaviour |
|-----------|-----------|
| `nmap` not installed | Exit with clear message pointing to install instructions |
| Scan returns no hosts | Exit cleanly: "No hosts found on <subnet>" |
| All hosts already known | Exit cleanly: "No new hosts found" |
| File write fails | Abort before any file is modified (tmp/replace strategy) |
| User presses Ctrl+C mid-loop | Exit cleanly, write nothing |
